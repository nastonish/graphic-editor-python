## Purpose of This Repository

This repository is a working repository for developing a student Python graphic editor project.

The final project should be a complete desktop graphic editor based on previous laboratory works about graphics programming.

The main project requirements are described in:

- `README.md`
- `PROJECT_REQUIREMENTS.md`
- `docs/labs_summary.md`
- reference PDFs in `reference_labs/`

The `reference_labs/` folder is only for understanding the original laboratory tasks. Do not depend on it in the application code.

---

## General Development Rules

1. Keep the project simple, clear, and suitable for a student course project.
2. Use Python with Tkinter for the graphical interface.
3. Use Pillow for image processing, opening and saving image files.
4. Do not add unnecessary external dependencies.
5. The final application must run with:

``bash
python main.py
Keep the code readable and easy to explain during a university defense.
Do not create overly complex architecture.
Do not split the project into too many files unless it is really useful.
Keep the project runnable after every major change.

Implementation Notes

Recommended approach:

Use a Pillow Image object as the main image storage.
Use Tkinter Canvas to display the current image.
Use ImageDraw for drawing operations.
Use temporary Canvas objects for previewing line, rectangle and ellipse while the mouse is moving.
Store full image snapshots for Undo.
Use canvas.canvasx(event.x) and canvas.canvasy(event.y) when working with scrollbars.
Use standard Tkinter dialogs:
filedialog
colorchooser
messagebox
Use the webbrowser module to open help.html.

## Lab 9: Graphic Operations

### Main Idea

Lab 9 continues Lab 8 and adds editing operations to the graphic editor.

The editor should not only draw figures, but also allow the user to cancel or edit operations.

### Important Requirements from Lab 9

Lab 9 includes three main groups of operations:

1. Canceling an unfinished drawing operation with the Esc key.
2. Undoing already completed operations.
3. Copying and moving separate figures or selected image areas.

---

### Canceling Current Operation with Esc

When the user starts drawing a figure and moves the mouse, the program shows a temporary preview.

If the user presses Esc before finishing the operation, the temporary figure should disappear and the operation should be canceled.

This applies to tools such as:

- Line
- Rectangle
- Ellipse
- Selection rectangle

### Undo Last Operation

The program should allow the user to cancel the last completed operation.

For raster graphics, the lab suggests saving previous image states or changed areas.

For this Python project, a simplified raster approach is acceptable:

- before each operation that changes the image, save a copy of the current image;
- when Undo is selected, restore the previous image state.

Undo should work for:

- drawing operations
- clearing the canvas
- paste operation
- filters
- rotation
- mirror operations

### Selection, Copy and Paste

Lab 9 requires adding a selection operation.

Basic expected behavior:

1. User chooses the selection tool.
2. User drags the mouse to select a rectangular area.
3. The selected area is copied with Copy.
4. The copied fragment can be inserted with Paste.
5. The inserted copy can be placed or moved if possible.

For the final project, a simple rectangular selection is enough.

### Large Images and Scrollbars

The editor should be able to work with images larger than the visible window.

This requires:

- horizontal scrollbar
- vertical scrollbar
- correct connection between the canvas and scrollbars
- correct drawing coordinates when the image is scrolled

In Tkinter, the coordinates should be calculated using:

```python
canvas.canvasx(event.x)
canvas.canvasy(event.y)

This is important because after scrolling, the mouse position on the screen is not the same as the real image coordinate.
