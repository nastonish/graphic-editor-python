# AGENTS.md

## Repository Purpose

This repository is a working repository for developing a student Python graphic editor project.

The final result should be a complete desktop graphic editor based on previous laboratory works from the graphics programming section of the system programming course.

The project should combine and extend the ideas from Lab 8, Lab 9 and Lab 10 into one finished application.

Main reference files in this repository:

- `README.md` — short project description;
- `PROJECT_REQUIREMENTS.md` — general project requirements;
- `docs/labs_summary.md` — summary of previous laboratory works;
- `reference_labs/` — original laboratory task files used only as reference materials.

The `reference_labs/` folder is only for understanding the previous laboratory works. The application code must not depend on files from this folder.

---

## Main Development Goal

Create a complete Python desktop application: a simple graphic editor.

The editor should allow the user to:

- create a new image;
- draw on a canvas;
- use basic drawing tools;
- edit selected areas;
- apply simple image operations;
- open and save image files;
- use help documentation;
- work with a clear and user-friendly interface.

The project should look like a finished student system programming project, not like a small temporary test script.

---

## Functional Requirements

The graphic editor should support the following groups of functions.

---

## File Operations

Implement:

- New image;
- Open image;
- Save image;
- Save image as;
- Exit.

The program should ask for confirmation before losing unsaved changes.

Confirmation is required before:

- creating a new image if the current image was modified;
- clearing the canvas;
- exiting the program if the current image was modified.

Use standard file dialogs for opening and saving images.

Supported formats should include at least:

- PNG;
- JPG/JPEG if possible;
- BMP if possible.

Saving as PNG is required.

---

## Drawing Tools

Implement the following tools:

- Pencil;
- Line;
- Rectangle;
- Ellipse/Oval.

The tools should work with mouse events:

- mouse button down;
- mouse movement;
- mouse button up.

For Line, Rectangle and Ellipse, show a temporary preview while the user drags the mouse.

When the mouse button is released, commit the final figure to the image.

---

## Drawing Parameters

The user should be able to choose:

- line color;
- fill color;
- line width.

Use a standard color chooser dialog for color selection.

Line width can be selected using a simple spinbox, combobox, slider, or similar control.

The current selected tool and parameters should be visible or easy to understand from the interface.

---

## Editing Operations

Implement:

- Undo last operation;
- Cancel current unfinished drawing operation with Esc;
- Clear canvas with confirmation;
- Rectangular selection;
- Copy selected area;
- Paste copied area;
- Delete or clear selected area if possible.

A simplified selection implementation is acceptable.

Expected selection behavior:

1. User chooses the selection tool.
2. User drags the mouse to select a rectangular area.
3. The program shows the selected rectangle.
4. User can copy the selected fragment.
5. User can paste the copied fragment.

If the user tries to copy without a selected area, show a clear message in Ukrainian.

Example:

```text
Спочатку виберіть область зображення.
```

If the user tries to undo when there is no previous action, show:

```text
Немає дії для скасування.
```

---

## Undo Requirements

Undo should work for the main operations that change the image.

Before each operation that changes the image, save a copy of the current image.

Undo should restore the previous image state.

Undo should work for:

- pencil drawing;
- line drawing;
- rectangle drawing;
- ellipse drawing;
- paste operation;
- clear canvas;
- filters;
- rotation;
- mirror operations.

A full image snapshot history is acceptable for this student project.

---

## Esc Cancel Requirements

Pressing Esc should cancel only the current unfinished operation.

Examples:

- unfinished line preview;
- unfinished rectangle preview;
- unfinished ellipse preview;
- unfinished selection rectangle.

Esc should remove the temporary preview object.

Esc should not undo already completed operations.

---

## Image Operations

Implement at least the following image processing operations:

- Grayscale filter;
- Color inversion;
- Rotate 90 degrees;
- Mirror horizontally;
- Mirror vertically.

If possible, add one additional simple operation:

- blur;
- sharpen;
- resize;
- brightness adjustment;
- insert text;
- draw grid;
- clear selected area.

The additional operation is optional but useful for a stronger final project.

---

## Large Image and Scrollbar Support

The editor should support images larger than the visible window.

Use:

- horizontal scrollbar;
- vertical scrollbar.

When handling mouse events on a scrolled canvas, use canvas coordinate conversion.

In Tkinter, this means using:

```python
canvas.canvasx(event.x)
canvas.canvasy(event.y)
```

This is important so that drawing works correctly after scrolling.

## Implementation Notes

Recommended implementation approach:

1. Use a Pillow `Image` object as the main internal image.
2. Use Tkinter `Canvas` to display the current image.
3. Use `ImageTk.PhotoImage` to show the Pillow image on the canvas.
4. Use `ImageDraw` for committing drawing operations to the image.
5. Use temporary canvas objects for previews.
6. Save previous image states for Undo.
7. Use `filedialog` for opening and saving.
8. Use `colorchooser` for choosing colors.
9. Use `messagebox` for confirmations and errors.
10. Use `webbrowser` for opening `help.html`.

For drawing previews:

- mouse down stores the start point;
- mouse move updates a temporary object;
- mouse up commits the shape to the Pillow image.

For pencil drawing:

- mouse down starts drawing;
- mouse move draws short line segments;
- mouse up finishes the operation.

For selection:

- selection should be rectangular;
- the selected area should be stored as coordinates;
- copy should crop the selected image fragment;
- paste should insert the copied fragment.
